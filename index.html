Here‚Äôs an exploratory data analysis (EDA) of the dataset, automatically generated using the ydata_profiling tool. The dataset, New York City Bus Data from Kaggle, includes four CSV files containing data from June, August, October, and December 2017. It consists of 17 columns and approximately 26 million entries, with data recorded every 10 minutes.

BUS TRAFFIC

Here‚Äôs a graphical representation of bus traffic in New York City. The lighter areas indicate regions with the most buses, not necessarily delays. The data represents a 10% sample of the December 2017 dataset.

The features are as follows:

  • You can view the map in 3D or 2D.
  • You can tilt, rotate, and zoom.
  • Legend can be displayed.
  • There are two layers:
    • The first, initially displayed, represents traffic using points on the map.
    • The second, initially hidden, represents traffic using hexagons for larger regions.
  • A filter allows you to observe traffic at specific times of the day (the filter is set to display all day by default).

BUS DELAYS

Here‚Äôs a graphical representation of bus delays in New York City. The darker the red color, the higher the delay. The data represents a 10% sample of the December 2017 dataset.

The features are as follows:

  • You can view the map in 3D or 2D.
  • You can tilt, rotate, zoom and select a line.
  • Legend can be displayed.
  • There are three layers:
    • The first, initially displayed, represents the delays using a heatmap.
    • The second, initially hidden, represents the delays using hexagons for larger regions Does not take into account the number of buses / traffic density.
    • The third, initially hidden, represents the areas with the most delays. Takes into account the number of buses / traffic density (zoomed in or out to show more or fewer clusters).
  • A filter allows you to observe the delays at specific times of the day (the filter is set to display all day by default).

PEAK HOURS BY BUS STOP

Here‚Äôs a graphical representation of peak hours by bus stop. This colormap indicates the average delays throughout the day at each stop. A lot of delay is shown in red, while being on time or early is displayed in dark green. The points are positioned according to their geographic coordinates, and their size on the map corresponds to a 20-meter radius in the real world.

This step includes the following features:

  • Zoom in/out
  • Colormap
  • Search bar
  • Map navigation
  • Display stop name on hover
  • Clickable stops

When we mention "clickable stops" among the features, it means that when zoomed in and selecting a specific stop, an event is triggered.

A popup opens and provides additional information about the stop:

  • Name
  • Bus lines passing through
  • Average delay
  • A chart showing delays by hour
